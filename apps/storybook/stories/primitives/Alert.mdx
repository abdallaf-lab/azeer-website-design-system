Block-level intent callout. **Five variants** locked: info / success /
warning / destructive / ai. The `ai` variant is dedicated (unlike Badge,
where AI is `accent` + Sparkles) — Alerts are content-heavy and need
distinct visual + iconography so users immediately know the content is
AI-generated.

## Variants

| Variant | Icon | Use |
| --- | --- | --- |
| `info` | `Info` | Neutral information — "Maintenance scheduled" |
| `success` | `CheckCircle2` | Positive confirmation — "Template approved" |
| `warning` | `TriangleAlert` | Caution — "Quota almost reached" |
| `destructive` | `AlertCircle` | Error — "Webhook failing" |
| `ai` | `Sparkles` | AI suggestion — "Drafted with AI" |

## Where to use Alert vs. other surfaces

Per the error-surface hierarchy:

| Need | Surface |
| --- | --- |
| Persistent app-state ("Trial expires in 3 days") | `Banner` (shell-level singleton — not yet built) |
| Ephemeral action feedback ("Saved", "Copied") | `Toast` |
| User must decide before continuing | `Dialog` / `ConfirmDialog` |
| Inline form-field validation | `FormField error` |
| **Content-level callout in a page or section** | **`Alert`** |

## Tokens used

| Token | Where |
| --- | --- |
| `--color-{intent}-bg-subtle` | Variant background |
| `--color-{intent}-border` | 1px intent border |
| `--color-{intent}-text` | Icon color (info / success / warning / destructive) |
| `--color-ai-icon` (= `--color-accent-brand`) | Sparkles color on `ai` variant |
| `--color-fg-default` | Title text (`text-label-md`) |
| `--color-fg-default` | Description text (`text-body-sm`) — neutral, not variant-tinted |
| `--radius-lg` (8 px) | Corner |
| `--color-state-hover` / `-active` | Dismiss button hover/active |

## A11y

Role split mirrors the Toast canon:

| Variant | ARIA role |
| --- | --- |
| `destructive` | `role="alert"` — announced immediately |
| `info` / `success` / `warning` / `ai` | `role="status"` — polite announcement |

Always-on signals:

- The icon is `aria-hidden="true"` — the title and/or description carry the message.
- Dismiss button is a real `<button>` with `aria-label="Dismiss"` (overridable).
- Color **plus** icon **plus** text label together signal intent — never color alone (WCAG 1.4.1).

## Forbidden

- Building a `Callout` primitive — `Alert` covers it
- Building a `Banner` from `Alert` — Banner is shell-level (a separate singleton primitive, coming in a later batch)
- Toasting critical messages — use `Alert` or `Dialog` (toasts can be missed)
- Variant-tinted body text — title/description stay `fg-default`; the variant signals via icon + border + bg
- White text on `--color-warning-fill` — Alert never uses the fill anyway (always bg-subtle)
- Custom intents outside the five — propose via RFC, never add ad-hoc
