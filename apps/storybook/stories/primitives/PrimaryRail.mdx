## Behavior

The rail has **three states**, two of them transient:

| State | Layout | Visual width | When |
| --- | --- | :-: | --- |
| Collapsed | In flow | 56 px | Default. Not hovered, not pinned. |
| Hover-expanded | **Overlay** (floats over content) | 196 px | Mouse is inside the rail; not pinned. Collapses on hover-out. |
| Pinned | In flow | 196 px | User clicked the pin button. Workspace shifts right to make room. |

**Hover-expand floats; pin pushes.** The hover-expand never shifts the
workspace — the rail's flow width stays at 56 px and the expanded panel
overlays the main content with `bg-canvas` + `shadow-elev-2` for depth.
Only when the user clicks the pin does the rail take up flow width and the
workspace shifts to accommodate. This keeps cursor work in the content area
free of accidental layout shifts.

The pin button lives at the top of the rail, **opposite the brand logo** in
a `justify-between` layout. It only shows when expanded.

```
┌──────────────────────────┐
│ [logo]            [pin] │   ← header: space-between
│                         │
│  [icon] Home            │   ← items: icon + label
│  [icon] Inbox       12  │
│  ...                    │
│                         │
│  [icon] Settings        │   ← footer slot
│  EN/AR                  │
│  [avatar] Sara Khan     │
└──────────────────────────┘
```

When collapsed, the same structure renders icon-only (40 × 40 squares) and
the pin button is hidden — the brand logo centers in the rail's 56 px.

## Locked anatomy

| Property | Value |
| --- | --- |
| Width — expanded | 196 px |
| Width — collapsed | 56 px |
| Surface | Chromeless — inherits `canvas`, no border, no shadow |
| Vertical layout | Top (logo + pin) → Body (modules) → Bottom (settings/profile/notifications) |
| z-index | `--z-nav` (20) |
| Transition | `width` over `--duration-base` |

Per Shell.md, the L1 is **the only navigation chrome** — DS forbids a
global topbar. Anything you'd put in a topbar (search, settings, profile,
workspace switch, notifications) lives in this rail's bottom cluster or
the HelpBubble.

## API

```tsx
<PrimaryRail
  logo={<WorkspaceSwitcher />}
  footer={<RailFooter />}        // see usePrimaryRail() below
>
  <PrimaryRailItem icon={Inbox} label="Inbox" active />
  <PrimaryRailItem icon={Users} label="Contacts" />
</PrimaryRail>
```

### PrimaryRail props

| Prop | Type | Notes |
| --- | --- | --- |
| `defaultPinned` | `boolean` | Uncontrolled — initial pin state. Default `false`. |
| `pinned` | `boolean` | Controlled pin state. Pair with `onPinChange`. |
| `onPinChange` | `(pinned: boolean) => void` | Fires when the user clicks the pin button. |
| `collapsed` | `boolean` | **Legacy override.** When provided, disables hover-pin behavior AND overlay floating; consumer owns collapse state entirely. |
| `logo` | `ReactNode` | Top slot (logo + workspace switcher) |
| `children` | `ReactNode` | Module nav items |
| `footer` | `ReactNode` | Bottom cluster |
| `ariaLabel` | `string` | `aria-label` on the nav landmark. Default `"Primary navigation"` |

### PrimaryRailItem props

| Prop | Type | Notes |
| --- | --- | --- |
| `icon` | `LucideIcon` | Required — 20 × 20 icon in the start container |
| `label` | `string` | Required — hidden when collapsed, but used as `title` for hover tooltip |
| `active` | `boolean` | Adds `aria-current="page"` + visual three-signal active state |
| `badge` | `number / string` | Notification count — inline Badge when expanded, corner overlay when collapsed |
| `trailing` | `ReactNode` | Custom right-side content; hidden when collapsed |
| `collapsed` | `boolean` | Override — normally reads from `usePrimaryRail()` context |
| `flipIconOnRtl` | `boolean` | Mirror the icon under RTL (use for directional icons) |

### Reading collapsed state — `usePrimaryRail()`

Items inside the rail automatically read the collapsed state via context.
Anything else inside the rail (language switcher, avatar button, custom
footer affordances) can read it via the `usePrimaryRail()` hook:

```tsx
import { usePrimaryRail } from "@azeer/ui";

function Footer() {
  const { collapsed, pinned } = usePrimaryRail();
  return collapsed
    ? <CompactProfile />
    : <FullProfile />;
}
```

### PrimaryRailItem

Active state per States.md §nav-item:

- Background → `accent-bg-subtle`
- Label → `accent-text`
- **2 px start-edge bar** in `accent-fill` via a positioned `<span>` (no content shift)

Three signals together for selection (WCAG 1.4.1).

```tsx
<PrimaryRailItem
  icon={Inbox}
  label="Inbox"
  active
  badge={12}
  onClick={() => navigate("/inbox")}
/>
```

#### Anatomy

The icon lives in a fixed 40 × 40 container anchored to the start of the
row, **identical in both collapsed and expanded states**. Only the label
+ trailing slot appear and disappear as the rail width animates. The icon
never moves — no horizontal jitter during the hover-expand transition.

#### `badge` vs `trailing`

| Prop | Expanded | Collapsed |
| --- | --- | --- |
| `badge` (number / short string) | Inline accent Badge at end of row | Small corner overlay on the icon |
| `trailing` (ReactNode) | Inline at end of row | Hidden |

Use `badge` for notification counts so the indicator survives the
collapsed state. Use `trailing` for any other custom right-side content
(presence dots, arrows, etc.) where the collapsed-state corner overlay
doesn't fit.

```tsx
{/* Notification count — visible in both states */}
<PrimaryRailItem icon={Inbox} label="Inbox" badge={12} />

{/* Custom trailing — visible only when expanded */}
<PrimaryRailItem icon={Settings} label="Settings" trailing={<KbdHint>⌘,</KbdHint>} />
```

`PrimaryRailItem` no longer needs `collapsed` in normal use — it reads
from context. Pass `collapsed` explicitly only if the item lives outside
a `PrimaryRail` (rare).

When collapsed, items render as 40 × 40 icon squares with `title={label}`
tooltip; only `badge` (as corner overlay) remains visible.

## Tokens used

| Token | Where |
| --- | --- |
| `--color-canvas` | Inherited surface; also the floating overlay's bg |
| `--shadow-elev-2` | Floating overlay depth |
| `--color-state-hover` | Item hover overlay |
| `--color-accent-bg-subtle` + `-accent-text` + `-accent-fill` | Active item bg + text + start bar |
| `--color-accent-bg-subtle` + `-accent-text` | Pinned-state pin button |
| `--color-accent-fill` + `-fg-on-accent` | Collapsed-state badge corner overlay |
| `--color-fg-default` | Resting item label |
| `--color-fg-muted` | Pin button (resting) |
| `--text-body-md` | Item typography |
| `--text-label-xs` | Corner badge numeric pill |
| `--radius-md` (item) | Corners |
| `--z-nav` (20) | Stacking tier |

## A11y

- `<nav aria-label="Primary navigation">` landmark.
- Active item carries `aria-current="page"` (auto-set by `active` prop).
- Pin button: `aria-label="Pin sidebar"` / `"Unpin sidebar"` + `aria-pressed={pinned}`.
- Collapsed items expose their label as `title` so hover tooltip works without losing the label semantically.
- Focus ring inherited from the global `:focus-visible` rule.

## RTL behavior

- Rail sits at the start side of the viewport — start in LTR, end (right) in RTL.
- Pin sits at the end of the header — flips to the start side under RTL via `justify-between` (logical).
- Active 2 px bar uses logical `start-0` — mirrors automatically.
- Pass `flipIconOnRtl` on directional icons (chevrons, arrows).

## Forbidden

- Global topbar paired with PrimaryRail — DS bans (this rail IS the chrome)
- Multiple `PrimaryRail` instances — singleton per AppShell
- Custom widths outside 196 / 56 — locked
- A "horizontal" rail variant — vertical only
- Search input inside the rail body — search lives in the bottom cluster or `⌘K` palette
- Custom pin affordance elsewhere (footer, separate toggle button) — the pin lives next to the logo, no exceptions
