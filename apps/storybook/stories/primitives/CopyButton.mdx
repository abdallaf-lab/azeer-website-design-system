Clipboard write + Toast feedback. The `Copy` icon flips to `Check` for
~2 seconds after a successful write so users get both inline visual
feedback and the ambient `toast.success` (3 s per DS canon).

Used in the Inspector for IDs / API keys / phone numbers / webhook URLs.

## API

```tsx
<CopyButton value="acct_8f3kdo10" successMessage="Account ID copied" />

// With visible label
<CopyButton value="…" variant="secondary" size="sm">
  Copy ID
</CopyButton>

// With description in the toast
<CopyButton
  value="ak_8f3kdo10xyzabcdef12345"
  successMessage="API key copied"
  successDescription="Don't share this with anyone."
/>
```

| Prop | Type | Notes |
| --- | --- | --- |
| `value` | `string` | Text written to the clipboard |
| `successMessage` | `string` | Toast title on success. Default `"Copied"` |
| `successDescription` | `string` | Toast description on success (optional) |
| `errorMessage` | `string` | Toast title on clipboard failure. Default `"Couldn't copy"` |
| `size` | `"icon-sm" \| "icon-md" \| "sm" \| "md"` | Default `"icon-sm"` |
| `variant` | `"ghost" \| "secondary"` | Default `"ghost"` |
| `children` | `ReactNode` | Optional text label next to the icon. When present, drop `aria-label` requirement |
| `ariaLabel` | `string` | Required when icon-only. Default `"Copy"` |
| All standard `Button` props (except `onClick`, `children` when text is overridden) | — | Pass through |

## Behavior

| Action | Result |
| --- | --- |
| Click | `navigator.clipboard.writeText(value)` |
| Success | Icon → `Check` for 2 s + `toast.success(successMessage)` |
| Failure (no clipboard permission, insecure context) | `toast.error(errorMessage)` |
| Re-click during the 2 s window | Resets the timer; icon stays `Check` |

## Tokens used

Inherits from `Button` ghost / secondary variants:

| Token | Where |
| --- | --- |
| `--color-state-hover` / `-active` | Hover / active overlay (ghost) |
| `--color-fg-default` | Icon color (resting + Check state) |
| `--duration-fast` + `--ease-standard` | Hover transition |

## A11y

- Icon-only: `aria-label="Copy"` (override via `ariaLabel`).
- With visible text label: button's accessible name comes from the text — `aria-label` is intentionally NOT set.
- Toast on success uses `role="status"` (polite) — SR users hear the confirmation without focus stealing.
- Clipboard failures (e.g., HTTP-served pages without `Clipboard API` permission) gracefully fall back to the error toast — no thrown exception bubbles to the app.

## Forbidden

- CopyButton for an action that *isn't* a copy (e.g., "copy to clipboard then navigate") — split into two actions
- Skipping the toast confirmation — clipboard writes are silent at the system level; the toast is the only feedback for screen-reader users
- Long values without `successDescription` — when the user can't see what they copied, echo it in the toast
- Multiple CopyButtons firing in rapid succession — each fires its own toast (max 3 visible per Toaster rules)
- Using `navigator.clipboard.writeText` directly in app code — wrap with CopyButton for the consistent UX
