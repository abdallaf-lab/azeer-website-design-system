## API

```tsx
const [page, setPage] = useState(1);

<Pagination
  page={page}              // 1-based
  pageCount={50}
  onPageChange={setPage}
  siblingCount={1}         // default — pages shown on each side of current
/>
```

| Prop | Type | Notes |
| --- | --- | --- |
| `page` | `number` | Current page (1-based) |
| `pageCount` | `number` | Total pages; Pagination renders `null` when ≤ 1 |
| `onPageChange` | `(page) => void` | Fires with the new page |
| `siblingCount` | `number` | Pages on each side of current. Default 1 |
| `disabled` | `boolean` | Disables all controls |
| `ariaLabel` | `string` | `aria-label` on the `<nav>`. Default `"Pagination"` |

## Algorithm

The page-button array follows the Intercom / Stripe / GitHub pattern:

- ≤ 7 pages total: show all
- Otherwise: `1 … {siblings around current} … {pageCount}`
- Ellipsis only appears when there's an actual gap (not for the page next to it)

For `siblingCount=1` (default) with `current=6` of 20 pages:

```
[‹] [1] [..] [5] [6] [7] [..] [20] [›]
```

For `siblingCount=2` the inner window widens to 5 pages.

## Tokens used

| Token | Where |
| --- | --- |
| `--color-accent-bg-subtle` + `-accent-text` + `-accent-border` | Active page button |
| `--color-state-hover` | Inactive page hover |
| `--color-fg-default` | Inactive page text |
| `--color-fg-muted` | Ellipsis + chevron icons |
| `--text-label-sm` | Page number typography |
| `--radius-md` | Button corners |

## A11y

- Wraps in a `<nav aria-label="Pagination">` — landmark semantic.
- Active page uses `aria-current="page"`.
- Prev / Next buttons have `aria-label="Previous page"` / `"Next page"`.
- Page buttons have `aria-label="Page N"`.
- Disabled state passes `disabled` attribute (excluded from focus order automatically).
- Ellipses are `aria-hidden` (decorative — they don't represent a page).

## RTL behavior

Chevrons use the Icon wrapper's `flipOnRtl` prop — point ← under
`dir="rtl"` so "previous" visually reads as start-side regardless of locale.

## Forbidden

- A jump-to-page input field — DS canon explicitly forbids this affordance
- Per-page-size dropdown inside the Pagination component — that's a separate control (place it adjacent if needed)
- Custom button colors per feature — single token set, system-wide
- More than ~50 visible page buttons — `siblingCount > 3` is hostile UX; use search/filter instead
- Hiding the component when `pageCount === 1` is automatic (returns `null`); don't wrap with extra conditional logic
