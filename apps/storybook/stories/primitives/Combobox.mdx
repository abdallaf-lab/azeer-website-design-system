## API

```tsx
import { Combobox, type ComboboxOption } from "@azeer/ui";

const channels: ComboboxOption[] = [
  { value: "whatsapp", label: "WhatsApp" },
  { value: "voice",    label: "Voice" },
  { value: "sms",      label: "SMS" },
];

<Combobox
  options={channels}
  value={value}
  onValueChange={setValue}
  placeholder="Pick a channel"
  searchPlaceholder="Search channels…"
  emptyMessage="No matches."
/>
```

| Prop | Type | Notes |
| --- | --- | --- |
| `options` | `ComboboxOption[]` | `{ value, label, group?, keywords?, disabled? }` |
| `value` / `defaultValue` | `string` | Selected value (controlled / uncontrolled) |
| `onValueChange` | `(value \| undefined) => void` | Fires `undefined` when the active option is deselected |
| `placeholder` | `string` | Trigger placeholder |
| `searchPlaceholder` | `string` | Search-input placeholder |
| `emptyMessage` | `ReactNode` | Body when no items match the query |
| `size` | `"sm" \| "md" \| "lg"` | Matches Input scale |
| `disabled` | `boolean` | Standard |

`Combobox` is **generic over `T extends string`** — pass a string-literal
union to get type-safe `onValueChange`.

## Filtering & groups

cmdk filters items by their `value` plus any `keywords`. If `label` is a
plain string, it's added to keywords automatically. For React-node labels,
pass `keywords` explicitly with the searchable plain text.

Groups: each option may carry `group: "Agents"`. All options sharing the
same group land under that uppercase-eyebrow heading, in declared order.
Options without a `group` fall into a single unlabeled group at the top.

## Tokens used

Inherits from `Popover` + `Command` + the Input state matrix:

| Token | Where |
| --- | --- |
| `--color-surface` + `-border-strong` | Trigger surface + border |
| `--color-border-focus` | Trigger focus + open border tint |
| `--color-surface-overlay` | Popover (and inner Command) background |
| `--color-accent-bg-subtle` + `-accent-text` | Highlighted item (cmdk's `data-[selected=true]`) |
| `--color-accent-text` | Check icon on the selected item |
| `--color-fg-subtle` | Placeholder when no selection |
| `--color-fg-muted` | Chevron + group eyebrow |
| `--text-body-sm` / `-body-md` | Item / trigger typography |
| `--text-label-xs` | Group heading typography |
| `--shadow-elev-2` | Popover shadow |
| `--z-popover` (100) | Floating layer |

## Keyboard

| Key | Behavior |
| --- | --- |
| `Enter` / `Space` on trigger | Open popover; search input auto-focuses |
| Typing | Filters list in real time (fuzzy match via cmdk) |
| `↑` / `↓` | Move selection within filtered list |
| `Enter` | Select highlighted; close popover |
| `Esc` | Close; return focus to trigger |
| Click trigger again | Re-open without losing selection |

## A11y

- Trigger is `role="combobox"` with `aria-expanded` + `aria-haspopup="listbox"`.
- `FormField` wires `aria-required` / `aria-describedby` / `aria-invalid` onto the trigger via the cloneElement pattern.
- Search input auto-focuses on open (cmdk default).
- Selected state communicated by **icon + text color**, not color alone (WCAG 1.4.1).

## RTL behavior

Trigger layout flips automatically — chevron sits on the logical end side
via `justify-between` and natural flow. Inside the popover, items align
start → end. Search icon stays on the start side of the input.

## Forbidden

- Combobox with ≤ 7 truly static options — use `Select`
- Type-ahead on a Combobox without cmdk filtering enabled — accept it; the whole point is search
- Ad-hoc Popover + cmdk composition — use `Combobox`
- Empty `value` of `""` as "no selection" — `undefined` is the no-selection signal; cmdk + Radix both treat `""` as a real value
- Combobox with no `emptyMessage` — silent zero-state is a bug; users won't know if filtering worked
