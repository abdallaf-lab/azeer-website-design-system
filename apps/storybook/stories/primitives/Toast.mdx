## Helper API

| Helper | Duration | Icon | Use |
| --- | --- | --- | --- |
| `toast.success(msg, opts?)` | **3 s** | `CheckCircle2` (success-text) | "Message sent", "Saved", "Copied" |
| `toast.info(msg, opts?)` | 4 s | `Info` (info-text) | "Synced in background" |
| `toast.warning(msg, opts?)` | 5 s | `TriangleAlert` (warning-text) | "Approaching quota" |
| `toast.error(msg, opts?)` | **Persistent** | `AlertCircle` (danger-text) | "Webhook returned 502 ŌĆö [Retry]" |
| `toast.loading(msg, opts?)` | Until dismissed | Sonner's spinner | "Sending broadcastŌĆ”" ŌĆö call `toast.dismiss(id)` when done |
| `toast.ai(msg, opts?)` | 4 s | `Sparkles` (ai-icon ŌĆö brand indigo) | "AI summary ready" |
| `toast.promise(p, msgs)` | auto | spinner ŌåÆ variant-icon | Async pattern ŌĆö transitions loading ŌåÆ success / error |
| `toast.dismiss(id?)` | ŌĆö | ŌĆö | Manual dismiss; no `id` dismisses all |

**Why errors are persistent.** For an enterprise CPaaS where a missed error
can mean a missed customer issue, persistent wins. The user can always
dismiss via the close button (always visible). The max-stack-3 rule keeps
the surface from piling up.

## Locked behavior (Toast.md canon)

| Parameter | Value |
| --- | --- |
| **Position** | `top-end` ŌĆö top-right LTR, top-left RTL (auto-mirrors via `<html dir>` observer) |
| **Max stack** | 3 visible ŌĆö older toasts auto-dismiss when a 4th fires |
| **Viewport offset** | 16 px |
| **Inter-toast gap** | 8 px |
| **Min width** | 360 px |
| **Max width** | 440 px |
| **Surface** | `bg-surface` (white) ŌĆö uniform across variants; intent comes from icon only |
| **Radius** | `rounded-lg` (8 px) |
| **Shadow** | `--shadow-elev-3` |
| **z-index** | `--z-toast` (1600) ŌĆö above modals, below call-bar |
| **Close button** | Always visible; click or swipe-to-dismiss |

## When NOT to use a Toast

Per Errors.md, reach for a different surface when:

| Scenario | Use |
| --- | --- |
| Form validation per-field | `FormField` `error` (inline) |
| User must decide before continuing | `Dialog` / `ConfirmDialog` |
| Persistent app-state (trial, maintenance) | `Banner` (later batch ŌĆö shell-level) |
| Critical irreversible action | `Dialog` |
| Inline contextual help | `Tooltip` or `HoverCard` |
| Progress with % or steps | `Progress` |

## A11y

- Sonner uses `role="status"` for success/info/warning/ai/loading (polite announcement) and `role="alert"` for error (immediate).
- Focus is NOT stolen; action button is reachable via keyboard tab cycle once a toast is present.
- `prefers-reduced-motion` collapses enter/exit animations via the global rule in tokens.css.
- Hover pauses the auto-dismiss timer (Sonner default) ŌĆö gives users time to read.
- Swipe-to-dismiss on touch devices (Sonner native).

## Forbidden

- `import { toast } from "sonner"` ŌĆö ESLint blocks; route through `@azeer/ui`
- Calling raw `sonner.toast(...)` ŌĆö must use one of the seven helpers
- `position` override ŌĆö DS-locked to top-end
- `duration` override per-call to extend a variant ŌĆö if you want a different duration you want a different variant
- Multiple `<Toaster />` instances ŌĆö mount exactly one at app root
- Two action buttons in one toast ŌĆö max one (use a Dialog if you need two)
- Background color per intent ŌĆö DS uses uniform `bg-surface`; icon carries the signal
- Toast for form validation ŌĆö use `FormField` inline error
- Toast for blocking decisions ŌĆö use `ConfirmDialog`
- Toast for ongoing state ("Trial expiring") ŌĆö use `Banner`
- Custom toast component / library ŌĆö Sonner via this wrapper only
