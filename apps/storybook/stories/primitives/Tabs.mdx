## Anatomy

```tsx
<Tabs defaultValue="open">
  <TabsList>
    <TabsTrigger value="open">Open</TabsTrigger>
    <TabsTrigger value="snoozed">Snoozed</TabsTrigger>
    <TabsTrigger value="closed">Closed</TabsTrigger>
  </TabsList>
  <TabsContent value="open">‚Ä¶</TabsContent>
  <TabsContent value="snoozed">‚Ä¶</TabsContent>
  <TabsContent value="closed">‚Ä¶</TabsContent>
</Tabs>
```

| Slot | Layout |
| --- | --- |
| `TabsList` | `h-10` inline-flex row + `border-b border-border-divider` (1 px) |
| `TabsTrigger` | full-height button, `text-label-md`, `border-b-2 border-transparent`, `-mb-px` to overlap list border |
| `TabsContent` | `mt-4` block; matched to its trigger by `value` |

## States (per States.md, Tab row)

| State | Mechanism |
| --- | --- |
| default (inactive) | `text-fg-muted`, transparent bottom border |
| **hover** (inactive) | Color shift to `text-fg-default`, underline stays at `border-divider` |
| **focus** | Global `:focus-visible` ring on the button (Radix renders a `<button>` trigger) |
| **selected** (`data-[state=active]`) | Underline becomes `border-accent-fill` (2 px) + label `text-accent-text` |
| disabled | `opacity-50` + `cursor-not-allowed` |

The 2 px underline + accent text together signal selection ‚Äî never color
alone (WCAG 1.4.1). Hover is a separate state that doesn't compete with
selection ‚Äî selected tabs don't darken on hover.

## Tokens used

| Token | Where |
| --- | --- |
| `--color-border-divider` | List bottom border |
| `--color-accent-fill` | Selected underline |
| `--color-accent-text` | Selected label color |
| `--color-fg-muted` | Inactive label color |
| `--color-fg-default` | Hover label color (inactive) |
| `--text-label-md` (14 / 16 / 600) | Tab label typography |
| `--duration-fast` + `--ease-standard` | Color transitions |
| `--focus-ring` (global) | Focus indicator |

## Keyboard

Radix wires the full WAI-ARIA tabs pattern:

| Key | Behavior |
| --- | --- |
| `Tab` | Moves focus IN/OUT of the tab list (single tab stop within the list) |
| `‚Üê` / `‚Üí` | Move between triggers (auto-mirror under RTL ‚Äî `‚Üê` becomes next in RTL) |
| `Home` / `End` | Jump to first / last tab |
| `Space` / `Enter` | Activate focused tab |

## A11y

- TabsList carries `role="tablist"`; each Trigger is `role="tab"` with `aria-selected` synced by Radix.
- TabsContent is `role="tabpanel"` with `aria-labelledby` pointing at its trigger.
- Single tab stop on the list ‚Äî keyboard users land on the active tab, arrow through siblings, then tab out into the panel.

## RTL behavior

Arrow keys auto-mirror via Radix's direction handling. Tab order in the list
reads start ‚Üí end in both LTR and RTL (the natural document direction). No
manual flip needed.

## Forbidden

- Pill / segmented / background-tinted tab style ‚Äî DS bans
- Tabs without `value` matching a content panel
- Mixed-size tabs in one list ‚Äî all triggers are h-10 (40 px), period
- Nesting Tabs inside Tabs more than 1 level ‚Äî re-design the surface
- Manual `aria-selected` ‚Äî Radix handles it
- `Tabs` for navigation between routes (use a router ‚Äî Tabs is for content switching, not URLs)
- More than ~7 tabs in one row ‚Äî switch to a sidebar or DropdownMenu
