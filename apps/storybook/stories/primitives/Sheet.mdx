Edge-anchored panel built on the same `@radix-ui/react-dialog` primitive as
`Dialog` — same accessibility plumbing, different positioning + animation.
Use Sheets when you want a context surface alongside the page (filters,
secondary details, mobile action picker) without taking over the screen the
way a centered Dialog does.

## Sides — locked

| Side | Anchored to | Slide-in direction |
| --- | --- | --- |
| `start` | inline-start edge of viewport | from left (LTR) / from right (RTL) |
| `end` (default) | inline-end edge | from right (LTR) / from left (RTL) |
| `bottom` | block-end edge | from bottom |

**No `top`** per Components.md — sheets are anchored below the visual
rhythm. For a top-anchored "Banner" surface, use the `Banner` primitive
(shell-level, coming in a later batch).

## Slots

Same shape as Dialog (`SheetHeader` / `SheetTitle` / `SheetDescription` /
`SheetBody` / `SheetFooter` / `SheetClose`), with slightly tighter padding:

| Slot | Padding |
| --- | --- |
| `SheetHeader` | `px-5 pt-5 pb-4` |
| `SheetBody` | `px-5 py-2` |
| `SheetFooter` | `px-5 pt-4 pb-5` |

Why the 4 px tighter pad vs Dialog? Sheets are narrower (max-w-md → 28rem)
and live at the edge — the slimmer rhythm reads cleaner.

## Tokens used

| Token | Where |
| --- | --- |
| `--color-surface` | Sheet background |
| `--color-border-default` | 1 px edge border (start/end on side sheets, top on bottom sheet) |
| `--color-backdrop-overlay` | Backdrop scrim (shared with Dialog) |
| `--radius-xl` (12 px) | Rounded only on the side facing the canvas (start sheet → `rounded-e-xl`, end sheet → `rounded-s-xl`, bottom sheet → `rounded-t-xl`) |
| `--shadow-elev-3` | Sheet shadow |
| `--z-modal` / `--z-modal-bd` | Same tiers as Dialog |

## A11y

Inherits all Dialog accessibility from Radix:

- Focus moves in on open, returns to trigger on close.
- `Esc` closes; outside-click closes.
- Body scroll locked while open.
- `aria-labelledby` → `SheetTitle`, `aria-describedby` → `SheetDescription`.
- **`SheetTitle` is required** — Radix logs a warning if missing.

## RTL behavior

`side="start"` and `side="end"` are logical — `start-0` / `end-0`
positioning + `border-s` / `border-e` + `rounded-s-xl` / `rounded-e-xl`
mirror under `dir="rtl"`. The slide animation uses physical names
(`slide-in-from-left`, etc) with `rtl:` overrides so motion mirrors too.

`side="bottom"` is direction-agnostic — same in LTR and RTL.

## Forbidden

- `side="top"` — DS forbids
- Sheet for confirmation flows — use `Dialog` or `ConfirmDialog`
- Sheet for ephemeral feedback — use `Toast`
- Side sheet wider than `max-w-md` (28 rem) — at that size you want a Dialog or a full route
- Bottom sheet taller than 80 vh — capped intentionally
- Multiple Sheets stacked — flatten the UI
