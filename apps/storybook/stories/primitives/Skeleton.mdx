Content-shaped placeholders for the initial data-fetch state. Per
Loading.md Rule 1, Skeletons must **match the shape of incoming content**
so the layout doesn't shift when data lands. Four primitives cover most
patterns; for anything else, compose them.

## The four primitives

```tsx
<Skeleton className="h-4 w-32" />     // generic ŌĆö pass dimensions
<SkeletonText lines={3} />             // text block (mixed widths)
<SkeletonAvatar size="md" />           // matches Avatar sizing
<SkeletonCard />                       // composed: avatar + 2 lines + tail
```

### Skeleton

Base building block. Pass `h-N w-N` via `className` or computed widths via
`style`. Default radius `md`; pass `rounded="full"` for avatar-shaped or
`rounded="sm"` for tiny indicators.

### SkeletonText

Stack of lines with **cycling widths** (100% ŌåÆ ~90% ŌåÆ ~75%) so 4+ lines
feel like real prose. The final line uses `lastLineWidth` (60/70/80) to
suggest paragraph ending.

| Prop | Default |
| --- | --- |
| `lines` | `3` |
| `lineHeight` | `"md"` (sm = h-3, md = h-4, lg = h-5) |
| `gap` | `"default"` (tight = gap-2, default = gap-3, loose = gap-4) |
| `lastLineWidth` | `70` |

### SkeletonAvatar

Six sizes matching the Avatar component (xsŌĆō2xl). Pass `shape="rounded"`
for organization avatars.

### SkeletonCard

`SkeletonAvatar` + tail (timestamp-shaped Skeleton) + `SkeletonText`. The
canonical conversation-row / contact-row / list-row shape.

## When to use Skeleton

Per Loading.md decision matrix:

| Situation | Loader |
| --- | --- |
| **Container waiting for content** | **Skeleton** Ō£ō |
| Action in progress, Ōēż 2 s | `Spinner` |
| Known duration OR multiple steps | `Progress` |

## A11y

- Generic `Skeleton` is `aria-hidden` ŌĆö the container above usually carries `aria-busy`.
- `SkeletonText` + `SkeletonCard` carry `role="status"` + `aria-label="Loading"` so screen readers announce the loading intent.
- `prefers-reduced-motion` collapses `animate-pulse` to a static state (Tailwind built-in respects the media query).

## Forbidden

- Skeleton for Ōēż 1 s loads ŌĆö creates flicker; just let the brief delay pass
- Skeleton during a re-fetch when content already exists ŌĆö show an inline `Spinner` at the top, keep existing rows visible
- Skeleton mixed with real content in the same list ŌĆö all or nothing
- Skeleton + Spinner overlaid on the same surface ŌĆö pick one
- Skeleton + Progress on the same surface ŌĆö pick one
- Skeleton for action feedback ŌĆö use `Spinner` inside the button (`Button loading` once Phase 5 ships)
- Animated skeletons with multiple effects (shimmer + slide + pulse) ŌĆö one subtle pulse only
- Skeleton sized differently than the content it'll replace ŌĆö defeats the layout-stability purpose
