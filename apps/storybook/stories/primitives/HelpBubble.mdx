## Locked anatomy

| Property | Value |
| --- | --- |
| Size | 44 × 44 px (`h-11 w-11`) |
| Position | `fixed bottom-4 end-4` (logical — mirrors RTL) |
| Surface | `accent-fill` background + `fg-on-accent` icon |
| Shadow | `--shadow-elev-2` |
| z-index | `--z-bubble` (60) — above banner, below popovers |
| Icon | `HelpCircle` default; override via `icon` prop |
| Notification dot | 10 × 10 px `danger-fill` circle with `ring-surface`, top-end corner |

Clicking opens a `Popover` (the same primitive used elsewhere). Default
popover side is `top` aligned to `end` so content opens above and aligned
with the bubble's end edge — minimal travel distance, never covers the
bubble.

## API

```tsx
import { HelpBubble } from "@azeer/ui";

<HelpBubble aria-label="Open help" hasNotification={hasNew}>
  {/* anything renders inside the Popover */}
  <h3>Need help?</h3>
  <p>Browse our docs or chat with support.</p>
  <Button size="sm">Start a chat</Button>
</HelpBubble>
```

| Prop | Type | Notes |
| --- | --- | --- |
| `icon` | `LucideIcon` | Default `HelpCircle`; common overrides — `MessageCircle` (chat), `BookOpen` (docs) |
| `aria-label` | `string` | **Required** — icon-only trigger needs an accessible name |
| `hasNotification` | `boolean` | Renders a red dot in the corner |
| `children` | `ReactNode` | Content inside the Popover |
| `side` / `align` / `sideOffset` | Popover passthroughs | Default `top` / `end` / `12 px` |
| `className` | `string` | Overrides on the trigger (e.g. `bottom-6 end-6` for different offset) |
| `contentClassName` | `string` | Override on the Popover content (e.g. wider `w-96`) |

## Tokens used

| Token | Where |
| --- | --- |
| `--color-accent-fill` / `-hover` / `-active` | Bubble background (resting / hover / press) |
| `--color-fg-on-accent` | Icon color |
| `--shadow-elev-2` | Bubble shadow |
| `--color-danger-fill` | Notification dot |
| `--color-surface` | Notification-dot ring (so it pops against the indigo bubble) |
| `--z-bubble` (60) | z-index — below popover so the opened panel sits above |

## A11y

- Trigger is a real `<button>` with required `aria-label`.
- Notification dot is `aria-hidden` — pair `hasNotification` with a meaningful change in `aria-label` or trigger an SR-announceable status elsewhere when content changes.
- Focus ring inherited from the global `:focus-visible` rule.
- Popover focus/restore/Esc-close all handled by Radix.

## RTL behavior

- Position uses logical `end-4` — bubble moves to the bottom-left under `dir="rtl"`.
- Notification dot uses logical `end-1` — corner mirrors too.
- Popover `align="end"` is logical — content anchored to the bubble's end edge in either direction.

## Forbidden

- Multiple `HelpBubble` instances mounted simultaneously — shell-level singleton
- Floating help affordance at any position other than bottom-end — DS locks the placement
- Wrapping the trigger with a custom `Popover` — use the HelpBubble component (it composes a Popover internally)
- Custom colors per-feature on the bubble — `accent-fill` only
- Off-44 px size — locked
- Mounting from feature code — `HelpBubble` belongs at the app root or `AppShell`-level surface
