## API

```tsx
const [q, setQ] = useState("");

<SearchInput
  value={q}
  onValueChange={setQ}
  onClear={() => analytics.track("search-cleared")}
  placeholder="Search inbox‚Ä¶"
  size="md"
/>
```

| Prop | Type | Notes |
| --- | --- | --- |
| `value` | `string` | Controlled value (`""` for empty) |
| `onValueChange` | `(value) => void` | Fires on every keystroke + when X clears |
| `onClear` | `() => void` | Fires after `onValueChange("")` when the user clicks X |
| `size` | `"sm" \| "md" \| "lg"` | Same scale as Input |
| `clearLabel` | `string` | `aria-label` for X button. Default `"Clear search"` |
| All standard `Input` props (except `type`, `value`, `onChange`) | ‚Äî | Pass through |

## Tokens used

Inherits from `Input` (focus, disabled, aria-invalid). Adds:

| Class | Why |
| --- | --- |
| `ps-9` (logical) | Reserves space for the leading icon |
| `pe-9` (logical, conditional) | Reserves space for the X button when value present |
| `start-3` / `end-2` (logical positioning) | Mirrors automatically under RTL |

Icon sizes: 14 px (per the locked icon scale). X clear button is a 20 √ó 20
square inside the input padding.

## A11y

- Renders `type="search"` ‚Äî surfaces native search semantics (Escape clears in some browsers).
- X clear button is a real `<button>` with `aria-label="Clear search"` (overridable via `clearLabel`).
- Leading Search icon is `aria-hidden="true"` and `pointer-events-none` ‚Äî purely decorative.
- For autocomplete dropdowns, use `Combobox` instead ‚Äî SearchInput is just the text field.

## RTL behavior

- Leading icon sits at logical `start` ‚Äî left in LTR, right in RTL.
- Clear button sits at logical `end` ‚Äî right in LTR, left in RTL.
- Input padding (`ps-9 pe-9`) mirrors automatically.

## Forbidden

- Custom `<Input>` with `<Search>` icon spread across the codebase ‚Äî extract to SearchInput
- SearchInput inside a `Combobox` trigger ‚Äî Combobox already provides its own search field
- Multiple SearchInputs on the same surface ‚Äî usually means filtering needs a multi-select, not a second search field
- SearchInput for typeahead-replacement features ‚Äî use `Combobox` (cmdk-backed filtering + result list)
