Built on `@radix-ui/react-avatar` for the image-load handshake (initials
fallback shows after a 600ms delay so a fast image arrives without a flicker;
fallback wins if the image errors). **Two shapes** encode entity type:
`circle` for people (default), `rounded` for organizations. **Six sizes**
locked to 20 / 24 / 32 / 40 / 48 / 64 px.

## Shape — entity-type signal

| Shape | Use |
| --- | --- |
| `circle` (default) | People — agents, contacts, customers |
| `rounded` (`--radius-lg`, 8px) | Organizations, workspaces, brands |

Distinguishing person-vs-org at a glance matters in dense inboxes — never
override shape arbitrarily.

## Sizes

| Size | Pixels | Use |
| --- | --- | --- |
| `xs` | 20 | Inline meta (Inspector row, assignee chip) |
| `sm` | 24 | Conversation rows, dense lists |
| `md` (default) | 32 | Sidebar user, conversation header |
| `lg` | 40 | Inspector hero, mid-prominence |
| `xl` | 48 | Profile preview popovers |
| `2xl` | 64 | Account / Settings page hero |

## Presence dot

Status indicator overlaid on the bottom-end corner. Mirrors under RTL via the
logical `end-0` utility — start-corner under `dir="rtl"`.

| Status | Color token |
| --- | --- |
| `online` | `--color-success-fill` |
| `away` | `--color-warning-fill` |
| `busy` | `--color-danger-fill` |
| `offline` | `--color-fg-subtle` |

A `ring-2 ring-surface` (white) creates a notch so the dot reads as a
separate element rather than colliding with the avatar's edge.

## AvatarGroup

Overlapping row with overflow chip. The group **overrides each child's size**
so visual rhythm is consistent — pass `size` to `AvatarGroup`, not to
individual `Avatar` children.

Logical `-space-x-2 rtl:space-x-reverse` flips the overlap direction in RTL,
so the leftmost avatar in LTR becomes the rightmost in RTL.

## Tokens used

| Token | Where |
| --- | --- |
| `--color-surface-sunken` | Fallback background |
| `--color-fg-default` | Initials text color |
| `--color-surface` | Presence-dot ring, AvatarGroup ring |
| `--color-success-fill` / `-warning-fill` / `-danger-fill` / `-fg-subtle` | Presence statuses |
| `--text-label-xs` through `--text-heading-sm` | Initials type per size |
| `--radius-full` | Circle shape |
| `--radius-lg` (8px) | Rounded shape |

## A11y

- `alt` is **required** — it's the accessible name for both image and fallback.
- Initials wrapper carries `aria-label={alt}` so screen readers announce the person's name, not just the letters.
- Presence dot is `role="img"` with `aria-label="Status: Online"` (etc.) — color isn't the only signal.
- AvatarGroup is `role="group"`; the overflow chip carries `aria-label="N more"` so screen readers count the hidden avatars.

## Forbidden

- Avatar without `alt` — the prop is required for a reason
- Custom shape values (e.g. `square`) — only `circle` / `rounded`
- Avatar size override inside `AvatarGroup` — the group dictates size
- Branded color presence dots — only the four statuses in the matrix
- `xs` Avatar for anything outside inline metadata — it's the floor, not a default
