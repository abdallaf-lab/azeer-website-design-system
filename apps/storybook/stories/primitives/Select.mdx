Native-feeling dropdown built on `@radix-ui/react-select`. **≤ 7 static
options** per Components.md — above that, reach for `Combobox` (type-ahead

## Slots

```tsx
<Select>
  <SelectTrigger size="md"><SelectValue placeholder="…" /></SelectTrigger>
  <SelectContent>
    <SelectGroup>
      <SelectLabel>Agents</SelectLabel>
      <SelectItem value="sara">Sara Khan</SelectItem>
    </SelectGroup>
    <SelectSeparator />
    <SelectItem value="other">Other</SelectItem>
  </SelectContent>
</Select>
```

| Slot | Role |
| --- | --- |
| `Select` | Radix Root — `defaultValue` / `value` / `onValueChange` / `disabled` / `name` (form integration) |
| `SelectTrigger` | The closed-state button. Looks like an Input + end-side chevron. |
| `SelectValue` | Renders the selected label inside the trigger. Use `placeholder` when no value. |
| `SelectContent` | Floating panel (portal-rendered) at `--z-popover`. Width auto-matches trigger width. |
| `SelectGroup` + `SelectLabel` | Semantic group with uppercase eyebrow header (`text-label-xs`) |
| `SelectItem` | Option row. Check icon appears on the end side when selected. |
| `SelectSeparator` | 1 px divider between groups |

## Sizes

| Size | Height | Type |
| --- | --- | --- |
| `sm` | 32 px | `text-body-sm` (13) |
| `md` (default) | 40 px | `text-body-md` (14) |
| `lg` | 48 px | `text-body-md` (14) |

Same scale + same state matrix as Input (focus → border-focus tint,
disabled → opacity 0.5 + sunken bg, aria-invalid → danger border).

## Tokens used

| Token | Where |
| --- | --- |
| `--color-surface` / `-surface-sunken` | Trigger bg (resting / disabled-or-readonly) |
| `--color-surface-overlay` | Content (dropdown) bg |
| `--color-border-strong` | Trigger border |
| `--color-border-focus` | Trigger focus border tint |
| `--color-fg-default` / `-fg-subtle` | Selected text / placeholder text |
| `--color-accent-bg-subtle` + `-accent-text` | Highlighted item (mouse and keyboard) |
| `--color-fg-muted` | Chevron + group label |
| `--text-body-sm` / `-body-md` | Trigger text by size |
| `--text-label-xs` | Group label (eyebrow) |
| `--radius-md` (trigger) / `-lg` (content) / `-sm` (items) | Shape tier |
| `--shadow-elev-2` | Content shadow |
| `--z-popover` (100) | Content z-index |

## Keyboard

Radix wires the full ARIA listbox pattern:

| Key | Behavior |
| --- | --- |
| `Space` / `Enter` (on trigger) | Open dropdown |
| `↑` / `↓` | Move highlight |
| `Home` / `End` | First / last item |
| `Enter` | Select highlighted, close |
| `Esc` | Close without selecting |
| `Tab` | Close + move focus to next field |
| `A`-`Z` | Type-ahead to first matching item |

## A11y

- Trigger is `role="combobox"` with `aria-expanded` and `aria-controls`.
- Content is `role="listbox"`; items are `role="option"` with `aria-selected`.
- Focus management handled by Radix — trigger focused on close, item focused on open.
- `FormField` wires `aria-required` / `aria-describedby` onto the trigger.

## Forbidden

- Select with > 7 options — use `Combobox`
- Native `<select>` — DS bans (no consistent styling, poor keyboard parity with custom Select)
- Custom dropdown built ad-hoc — always use `Select` or `Combobox`
- `value` of `""` (empty string) — Radix rejects; use a real "none" item with a distinct value
- Items without unique `value` — Radix throws
- Mixed sizes in the same form — `FormField`s use one size per form
