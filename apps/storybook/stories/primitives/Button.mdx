## Anatomy

- **Pill shape** (`rounded-full`) reserved for `primary` and `destructive` ‚Äî the highest-priority actions.
- **`rounded-md`** for `secondary` and `ghost` ‚Äî hierarchy encoded by shape, not just by color.
- **One `primary` per screen.** Two primaries = bug.
- **One size per row.** Mixed sizes break visual alignment ‚Äî ESLint can't catch this; code review does.

## Tokens used

| Token | Where |
| --- | --- |
| `--color-accent-fill` / `-hover` / `-active` | Primary background (fill-swap states) |
| `--color-fg-on-accent` | Primary text (white, 7.3:1 AAA) |
| `--color-surface` / `--color-surface-sunken` | Secondary resting / hover |
| `--color-border-strong` | Secondary outline |
| `--color-state-hover` / `--color-state-active` | Ghost overlay states |
| `--color-danger-fill` / `-hover` | Destructive (fill-swap) |
| `--color-fg-on-danger` | Destructive text (white, 4.6:1 AA) |
| `--text-label-sm` / `--text-label-md` | Type tokens by size |
| `--radius-md`, `--radius-full` | Shape by hierarchy |
| `--duration-fast`, `--ease-standard` | Hover transition |
| `--focus-ring` (global) | Focus indicator ‚Äî single canonical ring |
| `h-8` / `h-10` / `h-12` (= 32 / 40 / 48 px) | Locked control heights |

Zero hex / px / hard-coded values in the component source ‚Äî every style
resolves through Tailwind utilities backed by the `@theme` block.

## States

| State | Mechanism |
| --- | --- |
| **hover** | Fill-swap (primary, destructive) or overlay (secondary ‚Üí surface-sunken, ghost ‚Üí state-hover) |
| **focus** | Global `:focus-visible` rule applies the canonical 2px indigo ring at 2px offset |
| **active** | Fill-swap to `accent-fill-active` (primary), `state-active` overlay (ghost) |
| **disabled** | `opacity: 0.5` + native `disabled` + `cursor: not-allowed` (four signals together) |

## Theming axes ‚Äî toolbar toggles

The three theming axes set attributes on `<html>` via the global decorator:

- **Theme** ‚Äî `data-theme=light` (v1). Dark / high-contrast options flip the attribute but render unchanged today ‚Äî primitive overrides ship in v1.5.
- **Density** ‚Äî `data-density=comfortable` (v1). Compact is a v1.2 scaffold.
- **Direction** ‚Äî `dir=ltr` / `dir=rtl`. **Fully active in v1** ‚Äî flip the toggle and the Button's icon + label sides mirror through logical utilities.

## A11y

- Renders a real `<button>` (no `<div role="button">`).
- Icon-only sizes (`icon-sm` / `icon-md` / `icon-lg`) require an `aria-label` ‚Äî the stories above pass one.
- Disabled state pairs four signals (visual opacity, `pointer-events`, `aria-disabled`, cursor) so screen-reader and keyboard users get the same information as sighted mouse users.
- Focus ring is system-global; never re-rolled per component.

The Storybook A11y panel runs axe-core on every story ‚Äî open the **A11y**
tab in the addons panel to see live violations / passes / incomplete checks.

## Forbidden

- `variant="brand"` (deprecated alias) ‚Üí use `variant="primary"`
- `variant="link"` ‚Äî different element; use a `<Link>` / `<a className="text-accent-fill underline-offset-4 hover:underline">`
- `size="xs"` / `size="xl"` ‚Äî not in the allowlist
- `className="h-9"` (or other inline heights) ‚Äî use the `size` prop
- Passing `size="md"` explicitly ‚Äî it's the default; passing it is noise
- Two `primary` Buttons on the same screen
- Mixed sizes inside one row
