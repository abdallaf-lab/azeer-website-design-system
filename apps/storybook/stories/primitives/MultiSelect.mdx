Searchable multi-select. Same `Popover` + `Command` composition as
`Combobox`, with two visual differences:

1. **The trigger shows selected values as outline `Badge`s** with X-remove —
   not a comma-separated string.
2. **Each Command item has a checkbox indicator** on the start side
   (filled accent when selected, empty border otherwise).

Use for filter chips, agent assignment, tag selection, channel filtering
— anywhere multiple values can be picked at once from a searchable list.

## API

```tsx
import { MultiSelect, type ComboboxOption } from "@azeer/ui";

const tags: ComboboxOption[] = [
  { value: "vip",       label: "VIP",       group: "Priority" },
  { value: "important", label: "Important", group: "Priority" },
  { value: "english",   label: "English",   group: "Language" },
];

<MultiSelect
  options={tags}
  value={value}              // string[]
  onValueChange={setValue}   // (string[]) => void
  defaultValue={["vip"]}
  maxVisible={4}             // collapses beyond this to "+N more"
  placeholder="Apply tags…"
/>
```

| Prop | Type | Notes |
| --- | --- | --- |
| `options` | `ComboboxOption[]` | Same shape as `Combobox` |
| `value` / `defaultValue` | `string[]` | Array (controlled / uncontrolled) |
| `onValueChange` | `(string[]) => void` | Always fires with the full new array |
| `placeholder` | `string` | Trigger placeholder when nothing selected |
| `searchPlaceholder` | `string` | Inside the popover |
| `emptyMessage` | `ReactNode` | Zero-results body |
| `size` | `"sm" \| "md" \| "lg"` | Matches Combobox / Input |
| `maxVisible` | `number` | How many badges to show inline before `+N more` chip. Default 4. |
| `disabled` | `boolean` | Standard |

Generic over `T extends string` for typed values.

## Overflow / wrap behavior

Trigger uses `min-h-*` (not fixed `h-*`) so it grows vertically when many
values are selected. Beyond `maxVisible`, the remaining selections collapse
into a single `+N more` neutral badge — keeps the trigger height bounded.

## Removing a value

Each Badge in the trigger has its own X. Clicking the X:

1. Calls `onValueChange` with the value removed
2. **Doesn't open the popover** — the click is contained on the Badge

Inside the open popover, clicking an item toggles its inclusion (checkbox
flips).

## Tokens used

Inherits from `Combobox` + adds:

| Token | Where |
| --- | --- |
| `--color-accent-fill` + `--color-fg-on-accent` | Checked checkbox indicator (filled accent + white check) |
| `--color-border-strong` + `--color-surface` | Unchecked checkbox indicator |
| Outline `Badge` tokens (`--color-fg-default`, `--color-border-default`) | Selected-value badges in trigger |

## Keyboard

| Key | Behavior |
| --- | --- |
| All Combobox keybindings | Apply identically |
| `Enter` on an item | Toggles its inclusion (does NOT close the popover — multi-select stays open for further picks) |
| `Esc` | Closes the popover |

Note the difference from `Combobox`: Enter doesn't close the popover, so
users can pick several items in a row without re-opening.

## A11y

- Same `role="combobox"` + `aria-expanded` + `aria-haspopup="listbox"` on the trigger.
- Each Badge's X button has `aria-label="Remove {label}"` so screen-reader users can target specific removals.
- Checkbox indicators are `aria-hidden` (the row's `data-[selected=true]` carries the semantic).

## Forbidden

- Using MultiSelect for an enumeration that should be Single (RadioGroup or Combobox)
- Comma-string display of selections — use the Badge layout
- Overflow truncation without `+N more` indicator — the count signal is required so users know there's more
- MultiSelect with `value: ""` as "no selections" — pass `[]` (empty array) instead
- Triggering multi-select picks via Enter that *closes* the popover — Enter must keep it open here (different from Combobox)
