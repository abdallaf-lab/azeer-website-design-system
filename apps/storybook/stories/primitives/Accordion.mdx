## Type â€” what the spec lets you build

| Configuration | Behavior |
| --- | --- |
| `type="single"` | Only one item open at a time; opening closes the previous |
| `type="single" collapsible` | Same, plus all-closed is allowed (FAQ pattern) |
| `type="multiple"` | Any subset can be open simultaneously |

## Anatomy


## Animation

Content height transitions via Radix's `--radix-accordion-content-height`
CSS variable + two custom keyframes registered in tokens.css:

```css
@keyframes accordion-down {
  from { height: 0; }
  to   { height: var(--radix-accordion-content-height); }
}
@keyframes accordion-up {
  from { height: var(--radix-accordion-content-height); }
  to   { height: 0; }
}

@theme {
  --animate-accordion-down: accordion-down 200ms var(--ease-out);
  --animate-accordion-up:   accordion-up   150ms var(--ease-in);
}
```

The `animate-accordion-down` / `animate-accordion-up` utilities target
Radix's `data-[state=open]` / `data-[state=closed]`. `prefers-reduced-motion`
collapses both durations (global rule in tokens.css).

## A11y

- Each `AccordionTrigger` is a real `<button>` inside `<h3>` (Radix's default header wrapping).
- `aria-expanded` synced by Radix; `aria-controls` points at the matching content panel.
- Content panel is `role="region"` with `aria-labelledby` pointing at the trigger.
- `Esc` does not close (intentional â€” accordions are content, not modals).

## RTL behavior

Chevron sits on the logical end side automatically (`justify-between` lays
the row out start â†’ end, content first, chevron last). Mirror under
`dir="rtl"` to verify. Arrow-key navigation reads vertically â€” direction-
agnostic.

## Forbidden

- A custom `Collapsible` primitive â€” `Accordion type="single" collapsible` covers it
- A custom `Disclosure` primitive â€” same as Collapsible, banned
- Trigger styled as a pill / button â€” Accordion triggers are full-row, label-tier text
- Removing the chevron â€” it's the affordance that the item is expandable
- Animating something other than height (e.g. opacity-only) â€” the row needs to *grow* visibly so users see content arriving
- Accordion for navigation (links inside triggers that route away) â€” use Tabs or a sidebar
