A 1-pixel divider between sections or inline items. Horizontal by default.
Decorative by default — pass `decorative={false}` when the separator
semantically divides content groups and a screen reader should announce it.

## Tokens used

| Token | Where |
| --- | --- |
| `--color-border-divider` (`bg-border-divider`) | Divider color |
| `--alpha-divider` (0.06) | Base alpha — the divider color is derived from it |

The divider is intentionally lower-contrast (1.2 : 1 vs surface). It's
decorative by default and **WCAG-exempt** under 1.4.11 (decorative graphics
don't need 3 : 1 contrast).

## A11y

| `decorative` | ARIA output |
| --- | --- |
| `true` (default) | `role="none"` — hidden from the accessibility tree |
| `false` | `role="separator"` + `aria-orientation="horizontal"` (or `"vertical"`) — announced by screen readers |

Use semantic mode only when the divider conveys grouping that isn't already
inferable from layout. Most product surfaces use the decorative default.

## Forbidden

- `<hr style={{ … }} />` ad-hoc dividers — always use `Separator`
- Custom border colors per surface — divider color is one token, system-wide
- `decorative={false}` without nearby semantic content — be intentional about why a screen reader should announce it
- Vertical separator inside a container with no explicit height — `h-full` collapses to 0
- Mixing dividers with extra spacing on both sides — pick one rhythm signal (gap *or* divider), not both
