Maps a channel name to its locked brand color (`--color-channel-*`) and a
Lucide approximation of the channel's icon. Use inside `Badge`, `Avatar`
overlays, `ConversationListItem`, and any surface that needs to identify
which channel a message came from.

## Supported channels

| Channel | Color token | Notes |
| --- | --- | --- |
| `whatsapp` | `--color-channel-whatsapp` (`#25D366`) | White text fails 1.9 : 1 ‚Äî icon only |
| `voice` | `--color-channel-voice` (`#7B61FF`) | Same hex as brand indigo ‚Äî Voice IS Azeer |
| `sms` | `--color-channel-sms` (`#5B8DEF`) | Large text only with white |
| `email` | `--color-channel-email` (`#6B7280`) | Only channel passing 4.5 : 1 with white |
| `instagram` | `--color-channel-instagram` (`#E1306C`) | Icon only |
| `messenger` | `--color-channel-messenger` (`#0084FF`) | Icon only |
| `telegram` | `--color-channel-telegram` (`#229ED9`) | Icon only |

## Tokens used

| Token | Where |
| --- | --- |
| `--color-channel-{name}` | Icon `currentColor` via `text-channel-{name}` |
| Lucide `strokeWidth: 1.5` | Inherited from the system Icon wrapper |

The DS icon scale (12 / 14 / 16 / 20 / 24) governs the size prop ‚Äî no
arbitrary pixel values.

## A11y

| Mode | Output |
| --- | --- |
| Default | `aria-label` = channel display name (e.g. `"WhatsApp"`) |
| `decorative={true}` | `aria-hidden="true"` ‚Äî use when a sibling text label already names the channel |
| `label="‚Ä¶"` | Override the accessible name (e.g. `"Sent via WhatsApp"`) |

Always pair the icon with a text label when contrast on the icon's
saturated background would fail (mainly true for the Badge-with-icon
pattern under a colored fill).

## Forbidden

- ChannelIcon as a page background ‚Äî channel hexes are icon / badge fills only
- White text on a saturated channel fill ‚Äî most fail body contrast (see the table)
- A custom channel that isn't in the locked list ‚Äî propose via RFC, don't add a one-off color
- Using Lucide's brand icons module for channel marks ‚Äî that's licensed for editorial use, not app UI
- ChannelIcon without `aria-label` when the icon stands alone ‚Äî silently breaks screen readers
