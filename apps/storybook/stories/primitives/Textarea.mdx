## Rows + resize

- `rows={3}` by default — content area = 3 × 20 px line-height = 60 px, plus padding ≈ 80 px floor (`min-h-20`).
- `resize-y` lets the user drag-extend vertically only. Horizontal resize is locked — keeps the field aligned with the parent form column.
- For long-form bodies (descriptions, support replies), pass `rows={6}` or higher; users can always resize from there.

## State matrix

Identical to Input — see Primitives › Input docs for the table. State changes
happen via attributes (`disabled`, `readOnly`, `aria-invalid`) — never a
`variant` prop.

## A11y

- Always wrap in `FormField` (or pair with `Label` + `htmlFor`).
- `FormField` auto-wires `aria-describedby` to helper / error and `aria-invalid` when `error` is present.
- Maximum-length feedback: if you cap character count, show a `text-body-xs text-fg-muted` counter near the field (e.g. `262 / 280`); use `aria-describedby` to point at it.

## Forbidden

- Adding a `size` prop — there isn't one
- `resize: none` — locks the user out of an essential affordance; the system locks resize to `y` and that's the canon
- `resize: horizontal` or `resize: both` — break parent-column rhythm
- Raw `<textarea>` in app code
- Validating on every keystroke for length — let the user finish; show the counter as guidance
