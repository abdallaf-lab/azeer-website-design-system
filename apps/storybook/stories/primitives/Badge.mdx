## Variants

| Variant | Use |
| --- | --- |
| `neutral` | Generic metadata тАФ channel labels, counts, role tags |
| `info` | Informational status тАФ "Beta", "New" |
| `success` | Positive тАФ "Active", "Verified", "Online" |
| `warning` | Needs attention тАФ "Pending", "High priority" |
| `destructive` | Negative тАФ "Failed", "Blocked", "Suspended" |
| `accent` | Brand emphasis тАФ "Pro", "VIP", AI labels (pair with `<Sparkles />` icon prefix) |
| `outline` | Structural label тАФ filter chips, neutral tags |

## Sizes

`sm` (h-5 = 20 px) ┬╖ `md` (h-6 = 24 px, default). **No `lg`** тАФ the
metadata tier doesn't earn a hero size. If you want a bigger surface
you probably want an `Alert`, not a Badge.

## Tokens used

| Token | Where |
| --- | --- |
| `--color-*-bg-subtle` | Variant background (success, danger, warning, info, accent) |
| `--color-*-text` | Variant text color |
| `--color-surface-sunken` + `--color-border-default` | Neutral variant |
| `--color-border-default` | Outline variant |
| `--text-body-xs` (12 / 16 / 400) | Label typography |
| `--radius-md` | Default shape |
| `--color-state-hover` / `-active` | Remove button hover/active overlay |
| `--duration-fast` + `--ease-standard` | Remove button transition |

Zero hex in source тАФ every Badge color routes through a semantic token.

## States

The Badge itself has no interaction states тАФ it's not interactive. The
`removable` X-button has its own hover/focus states (overlay + canonical
focus ring inherited from `tokens.css`).

## A11y

- Renders a `<span>` (semantic neutral тАФ Badge is decorative metadata, not interactive).
- The remove button is a real `<button>` and requires `aria-label`. Default `"Remove"`; override per usage when context helps screen-reader users (e.g. `"Remove WhatsApp filter"`).
- Color **and** text together signal intent тАФ never color alone (WCAG 1.4.1).
- Channel hexes (saturated brand marks) are allowed as Badge background only when the Badge contains an icon, not text тАФ the channel name + neutral surface combo passes contrast where the saturated fill alone wouldn't.

## RTL behavior

The remove X sits on the **end** side via `-me-1 ms-0.5` (logical
margins) тАФ auto-mirrors under `dir="rtl"`. Flip the toolbar to RTL on the
**Removable** / **Filter chips** stories above to verify.

## Forbidden

- Separate `Tag` / `Chip` primitive тАФ use `<Badge removable>` instead
- Separate `Pill` primitive тАФ apply `rounded-full` at usage when needed
- Separate `ai` variant тАФ use `accent` + `<Sparkles />` icon prefix
- Channel hexes as page backgrounds тАФ icon / badge-with-icon only
- Badge size `lg` тАФ not in the allowlist
- Inline `style={{ background, color }}` тАФ always use a variant
