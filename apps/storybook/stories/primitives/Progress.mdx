## Determinate vs indeterminate

| `value` | Behavior |
| --- | --- |
| Number 0‚Äì100 | Determinate ‚Äî bar fills to the percentage; transitions smoothly |
| `null` (or `undefined`) | Indeterminate ‚Äî bar shows ‚Öì-width pulse animation, no percentage |

Pass `null` when you know the operation is in flight but the duration isn't
measurable. For ‚â§ 2 s actions of unknown duration, prefer `Spinner` ‚Äî the
indeterminate Progress sits between "fast Spinner" and "measurable Progress".

## Intents

| Intent | Color | Use |
| --- | --- | --- |
| `primary` (default) | `accent-fill` indigo | General progress (uploads, broadcasts, generic steps) |
| `success` | `success-fill` green | Reached 100% / completed |
| `warning` | `warning-fill` amber | Approaching capacity (quota > 80%) |
| `destructive` | `danger-fill` red | Exceeded (quota > 95%) |

## Tokens used

| Token | Where |
| --- | --- |
| `--color-surface-sunken` | Track background |
| `--color-accent-fill` / `-success-fill` / `-warning-fill` / `-danger-fill` | Bar fill per intent |
| `--radius-full` | Track + bar shape |
| `--duration-base` + `--ease-standard` | Width transition |

## A11y

- Radix sets `role="progressbar"` + `aria-valuenow` (when determinate) + `aria-valuemax="100"`.
- For indeterminate (`value === null`), Radix omits `aria-valuenow` ‚Äî screen readers announce as "indeterminate progress bar."
- Pair with visible numeric label when % is meaningful ‚Äî see the **Quota indicator** and **Live upload** stories.
- `prefers-reduced-motion` collapses the determinate width transition; indeterminate `animate-pulse` also static under reduced motion.

## When to escalate to Toast / Banner / Dialog

| Pattern | Surface |
| --- | --- |
| Inline progress (upload row, settings save) | **Progress** here |
| Big multi-step background job (broadcast send) | **Dialog with Progress** + cancel button |
| Persistent quota approaching | **Banner** with embedded Progress |

## Forbidden

- Progress for ‚â§ 2 s actions ‚Äî use Spinner
- Progress without percentage label when duration IS known ‚Äî defeats the purpose
- Indeterminate Progress when a real % is computable ‚Äî show the %
- Stale Progress that doesn't update for > 500 ms ‚Äî refresh every 500 ms or switch to indeterminate
- Custom bar colors per app feature ‚Äî locked to the four intents
- Animating `box-shadow` for progress (browser perf) ‚Äî `transform: translateX` is the correct path (Radix does this)
- Mixed loaders on the same surface ‚Äî Progress alone, never with Spinner or Skeleton
