`cmdk` inside a `Dialog`. The canonical global-action surface — search,
navigation, common commands. Mounts a global `keydown` listener for the
DS-locked `⌘K` / `Ctrl+K` shortcut (per Accessibility.md
§locked-keyboard-shortcuts).

Compose with the existing `Command` primitives — CommandPalette wraps
them in the modal chrome and shortcut wiring.

## API

```tsx
const [open, setOpen] = useState(false);

<CommandPalette open={open} onOpenChange={setOpen}>
  <CommandInput placeholder="Type a command or search…" />
  <CommandList>
    <CommandEmpty>No results.</CommandEmpty>
    <CommandGroup heading="Actions">
      <CommandItem onSelect={() => { … }}>
        <Icon icon={Reply} size={14} />
        Reply
        <span className="ms-auto"><Kbd>R</Kbd></span>
      </CommandItem>
    </CommandGroup>
    <CommandSeparator />
    <CommandGroup heading="Navigation">
      <CommandItem onSelect={() => navigate("/inbox")}>
        Go to Inbox
      </CommandItem>
    </CommandGroup>
  </CommandList>
</CommandPalette>
```

| Prop | Type | Notes |
| --- | --- | --- |
| `open` / `defaultOpen` / `onOpenChange` | Standard controlled-open trio | — |
| `enableGlobalShortcut` | `boolean` | Default `true`. Mounts `⌘K` / `Ctrl+K` listener on `document` |
| `size` | `"sm" \| "md" \| "lg"` | Dialog size — `"md"` (640 px) default |
| `children` | `ReactNode` | Compose with existing `Command*` primitives |

## The ⌘K shortcut

Listens on `document.keydown` for `(Cmd \| Ctrl) + K`. Toggles open / close
regardless of focus location (works while typing in form fields too,
which is the expected command-palette UX). To disable per-palette, pass
`enableGlobalShortcut={false}`.

For mounting the shortcut without rendering the palette (e.g., the parent
manages open state across multiple surfaces), use the exported hook:

```tsx
import { useCommandPaletteShortcut } from "@azeer/ui";

useCommandPaletteShortcut(() => setOpen((o) => !o));
```

## Anatomy

Internally:

```tsx
<Dialog open={isOpen} onOpenChange={setOpen}>
  <DialogContent size={size} hideCloseButton className="p-0">
    <Command>{children}</Command>
  </DialogContent>
</Dialog>
```

- `hideCloseButton` — palettes use `Esc` for dismissal, not an X
- `p-0` on the Dialog — the inner `Command` provides its own padding
- Dialog inherits all DS-locked sizing (sm 480 / md 640 / lg 800) and
  centering / focus-trap / backdrop behavior

## Tokens used

Inherits from `Dialog` (modal centering, `--elev-3`, backdrop scrim) and
`Command` (highlight = `accent-bg-subtle` + `accent-text`, uppercase
eyebrow group labels, mono shortcuts).

## Keyboard

| Key | Behavior |
| --- | --- |
| `⌘K` / `Ctrl+K` (global) | Toggle palette open / close |
| `Esc` (palette open) | Close (Radix Dialog default) |
| `↑` / `↓` | Move highlight |
| Typing | Filter list in real time (cmdk fuzzy match) |
| `Enter` | Run highlighted command; usually `onSelect` calls `setOpen(false)` |

## A11y

- Dialog focus-trap + `aria-modal` + Esc to close — all inherited from `Dialog`.
- Search input auto-focuses on open (cmdk default).
- Items are `role="option"`; the list is `role="listbox"`.
- Shortcut hints rendered with `Kbd` carry `dir="ltr"` so modifier symbols read correctly inside Arabic prose.

## Forbidden

- Multiple CommandPalette mounts competing for `⌘K` — only the most recent listener wins (last-in-first-priority by `addEventListener` semantics); make sure only one palette is mounted at a time
- Custom shortcut other than `⌘K` / `Ctrl+K` — DS-locked in Accessibility.md
- Critical actions only available via the palette — palette is a *convenience layer*, every action must also have a visible UI path
- Async-loaded items without a `loading` indicator — wrap with cmdk's `<Command.Loading>` or show a `SkeletonText` inside the list
- Replacing the Dialog backdrop with custom scrim — `Dialog` provides the canonical `--alpha-backdrop` 40% black
