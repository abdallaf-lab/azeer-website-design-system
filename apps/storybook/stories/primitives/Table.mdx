Two layers:

1. **Table primitives** â€” semantic `<table>` HTML with DS chrome. Use
   directly for static / simple tables.
2. **`DataTable`** â€” TanStack Table renderer. Pass a `useReactTable`
   instance; DataTable renders it through the Table primitives. Consumer
   owns all sorting / filtering / pagination / row-selection logic.

This hybrid (TanStack handles state and logic, Shadcn-style primitives
handle UI) is the recommended split â€” keeps the renderer dumb and tiny
while the engine handles the heavy lifting.

## DataTable composition

```tsx
import { useReactTable, getCoreRowModel, getSortedRowModel, â€¦ } from "@tanstack/react-table";
import { DataTable } from "@azeer/ui";

const table = useReactTable({
  data,
  columns,
  state: { sorting },
  onSortingChange: setSorting,
  getCoreRowModel: getCoreRowModel(),
  getSortedRowModel: getSortedRowModel(),
});

return <DataTable table={table} />;
```

DataTable is a renderer â€” **all** TanStack features (sorting, filtering,
pagination, row selection, grouping, expanding, column visibility, column
ordering) work because they're driven by the `table` instance the consumer
constructs.

For sortable columns, the canonical `columnDef.header` recipe:

```tsx
header: ({ column }) => (
  <Button
    variant="ghost"
    size="sm"
    onClick={() => column.toggleSorting(column.getIsSorted() === "asc")}
  >
    Name <Icon icon={ArrowUpDown} size={14} />
  </Button>
)
```

For row selection, drop a checkbox column at the start:

```tsx
{
  id: "select",
  header: ({ table }) => (
    <Checkbox
      checked={table.getIsAllPageRowsSelected()}
      onCheckedChange={(v) => table.toggleAllPageRowsSelected(!!v)}
    />
  ),
  cell: ({ row }) => (
    <Checkbox
      checked={row.getIsSelected()}
      onCheckedChange={(v) => row.toggleSelected(!!v)}
    />
  ),
}
```

## A11y

- Renders semantic `<table>` / `<thead>` / `<tbody>` / `<tr>` / `<th>` / `<td>` â€” screen readers parse it as a data table.
- Use `<TableCaption>` for a programmatic summary when the table's purpose isn't obvious from surrounding context.
- Sortable column headers should render an `<button>` (via `Button variant="ghost"`) so they're keyboard-focusable.
- Row checkboxes need `aria-label="Select row"` â€” the stories above pass labels.

## RTL behavior

- `text-start` on `<th>` and `align-middle` on cells flow naturally in both directions.
- `tabular-nums` on numeric cells (e.g. `Unread` count) keeps digits column-aligned regardless of direction.
- `text-end` is logical (`text-align: end`) â€” last column right-aligned in LTR, left-aligned in RTL.

## Forbidden

- `<table>` ad-hoc in app code â€” always use Table primitives
- Custom row colors per-feature â€” `state-hover` / `accent-bg-subtle` are the only allowed row tints
- Off-`h-14` row heights â€” pick `h-14` (default) or pass `h-10` for dense-table mode (per Spacing canon Â§dense-table)
- Sticky-position tweaks ad-hoc â€” the header is `bg-surface-sunken` so you can apply `sticky top-0` at the consumer level
- Loading a huge dataset without virtualization â€” use `@tanstack/react-virtual` alongside DataTable when row count > ~200
