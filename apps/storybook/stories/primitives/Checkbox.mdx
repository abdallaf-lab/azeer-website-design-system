Multi-select toggle built on `@radix-ui/react-checkbox`. Single canonical
size 16 × 16 px. Supports a third **indeterminate** state for select-all
patterns (parent checkbox in a tree where some — but not all — children are
checked).

## Three states

| State | Visual |
| --- | --- |
| unchecked | 1 px `border-strong` outline, `bg-surface` |
| **checked** | `bg-accent-fill` + `border-accent-fill` + white `Check` icon |
| **indeterminate** | `bg-accent-fill` + `border-accent-fill` + white `Minus` icon (a horizontal bar) |
| disabled (any state) | `opacity: 0.5` + `cursor: not-allowed` |

Use `indeterminate` to communicate "some, not all" — typical for a parent
"All channels" checkbox sitting above a list of channel rows. See the
**Select-all pattern** story above.

## Tokens used

| Token | Where |
| --- | --- |
| `--color-border-strong` | Unchecked outline |
| `--color-surface` | Unchecked background |
| `--color-accent-fill` | Checked / indeterminate bg + border |
| `--color-fg-on-accent` | Check / Minus icon color (white) |
| `--radius-sm` (4 px) | Corner — small radius for the small surface |
| `--duration-fast` + `--ease-standard` | State-change transitions |
| `--focus-ring` (global) | Focus indicator (Radix renders a button — covered) |

## Anatomy — icon switching

The Check + Minus icons coexist inside the indicator. Visibility flips via
`group-data-[state=…]` selectors on the Root, so the same component handles
controlled + uncontrolled instances without a React state branch:

```tsx
<Checkbox.Root className="group …">
  <Checkbox.Indicator>
    <Check className="group-data-[state=indeterminate]:hidden" />
    <Minus className="hidden group-data-[state=indeterminate]:block" />
  </Checkbox.Indicator>
</Checkbox.Root>
```

## A11y

- Renders a `role="checkbox"` element (a styled `<button>` under the hood — Radix). Native `<input type="checkbox">` semantics carry through.
- `aria-checked` is synced by Radix (`true` / `false` / `"mixed"` for indeterminate).
- Click target is 16 × 16 px — below the 24 × 24 WCAG 2.5.5 floor on its own, so **always pair with a clickable label** so the label text extends the hit area. Stories above show both `<label>` wrap and `Label` + `htmlFor` patterns.
- Disabled with a recoverable cause: explain why via `FormField` `helper`.

## RTL behavior

Checkbox is symmetric — no directional content. Layout flows from the
wrapping `<label>` or container; inside `RadioGroup` / a checklist, the
indicator stays on the start side and the text on the end side automatically
via the parent's `flex` direction.

## Forbidden

- `<input type="checkbox">` ad-hoc — always use `Checkbox`
- A `size` prop — there isn't one (per Variants.md, toggles are single-size)
- Bare Checkbox without a label or `aria-label` — fails 16 × 16 hit-area minimum without label-extension
- Using indeterminate as a third "maybe" state in user input — it's reserved for "some of, not all of" tree-state communication
- Custom checked colors per feature — `accent-fill` checked, period
- Stacking a `Switch` and `Checkbox` for the same control — pick by semantic: Switch = state, Checkbox = multi-select
