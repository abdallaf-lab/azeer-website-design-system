## Locked anatomy

| Property | Value |
| --- | --- |
| Height | **64 px** (`h-16`) per Shell.md |
| Padding | `px-5` |
| Title | `text-heading-md` (18 / 24 / 600), `<h2>` semantic |
| Meta | `text-body-sm` + `text-fg-muted` |
| Action group gap | `gap-3` |
| Bottom border | `border-b border-border-default` |
| Surface | `bg-surface` (same as Main card) |

## API

```tsx
<ModuleHeader
  title="Inbox"
  meta="12 open conversations"
  leading={<BackButton />}
  actions={
    <>
      <Button variant="secondary" size="sm">Filter</Button>
      <Button size="sm">Compose</Button>
    </>
  }
/>
```

| Slot | Use |
| --- | --- |
| `leading` | Back button, breadcrumb, leading avatar |
| `title` | Module name. Can be a ReactNode (badge inline, icon, etc.) |
| `meta` | Sub-line under the title (counts, status, version) |
| `actions` | Right-aligned action group — typically `Button size="sm"`s |

## Tokens used

| Token | Where |
| --- | --- |
| `--color-surface` | Background |
| `--color-border-default` | Bottom border |
| `--color-fg-default` | Title text |
| `--color-fg-muted` | Meta text |
| `--text-heading-md` | Title typography |
| `--text-body-sm` | Meta typography |

## A11y

- Renders `<header>` landmark with `<h2>` title — preserves heading hierarchy.
- For multi-page modules, the title should match the route's `<title>` (or `document.title`) so screen-reader landmark navigation matches browser tab labels.
- Action buttons inherit `sm` button focus ring from the global rule.

## RTL behavior

- `justify-between` flows naturally — title at inline-start, actions at inline-end.
- `truncate` on long titles uses end-side ellipsis in LTR, start-side in RTL (browser-native, automatic).
- `gap-3` between leading / title / actions is direction-neutral.

## Forbidden

- ModuleHeader at viewport-fixed position — it lives inside Main card
- ModuleHeader spanning L1 — never; it's per-module, inside the card
- Title typography override — locked `text-heading-md`
- Height override — locked `h-16`
- Multiple ModuleHeaders stacked — one per Main pane
- Inline navigation tabs inside ModuleHeader — use `Tabs` *below* the header inside the same Main card
