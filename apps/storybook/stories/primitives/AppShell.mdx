## Variants

### Standard (3-zone)

```
+-----+----------+--------------------+
| L1  | L2       | Main                |
| 196 | 230      | flex                |
+-----+----------+--------------------+
```

Used by: **Knowledge** · **Reports** · **Outbound** · **Contacts** · **Fin AI Agent** · **Settings** — and any future module.

### Object detail (5-zone, Inbox only)

```
+-----+--------+--------+------------+--------+
| L1  | Folders| List   | Main       | Details|
| 196 | 200    | 280    | flex       | 320    |
+-----+--------+--------+------------+--------+
```

Used by: **Inbox only.**

**🚫 Don't invent a 4-zone variant** — use A or B.

## API

```tsx
<AppShell
  variant="standard"
  primaryRail={<PrimaryRail …/>}
  sidebar={<Sidebar …/>}                  // standard
  banner={<Banner …/>}                     // optional
  helpBubble={<HelpBubble …/>}             // optional
>
  <ModuleHeader title="Inbox" />
  <ModuleBody />
</AppShell>

// Or — Inbox:
<AppShell
  variant="objectDetail"
  primaryRail={<PrimaryRail …/>}
  folders={<ShellPanel width={200}>…</ShellPanel>}
  list={<ShellPanel width={280}>…</ShellPanel>}
  details={<ShellPanel width={320}>…</ShellPanel>}
>
  <ModuleHeader title="Conversation" />
  <ConversationView />
</AppShell>
```

| Prop | Type | Notes |
| --- | --- | --- |
| `variant` | `"standard"` (default) / `"objectDetail"` | DS-locked — no other values |
| `primaryRail` | `ReactNode` | The L1 rail — required for both variants |
| `sidebar` | `ReactNode` | Standard variant L2 |
| `folders` | `ReactNode` | ObjectDetail folders panel (200 px) |
| `list` | `ReactNode` | ObjectDetail conversation list (280 px) |
| `details` | `ReactNode` | ObjectDetail details panel (320 px) |
| `banner` | `ReactNode` | Trial / maintenance / system banner. Above body row, NOT spanning L1. |
| `helpBubble` | `ReactNode` | Fixed bottom-end help affordance |
| `children` | `ReactNode` | Main pane content (typically `ModuleHeader` + module body) |

## ShellPanel

Helper for the three `objectDetail` narrower panels — 200 / 280 / 320 px
locked widths.

```tsx
<ShellPanel width={200} ariaLabel="Folders">
  <SidebarItem … />
</ShellPanel>
```

Standard variant uses `Sidebar` (230 px) — not `ShellPanel`.

## Locked dimensions (Shell.md)

| Element | Value |
| --- | --- |
| L1 rail expanded | 196 px |
| L1 rail collapsed | 56 px |
| L2 sub-nav | 230 px |
| Inbox folders | 200 px |
| Conversation list | 280 px |
| Details panel | 320 px |
| Shell gutter (between all panels) | 4 px |
| Trial banner | 48 px |
| HelpBubble | 44 × 44 |

**All values are multiples of 4** — the spacing rhythm holds across the
entire chassis.

## Why the banner sits inside the workspace column

Per Shell.md: the banner is **content-level** — inside the workspace
column above the body row. It does **not span the L1 rail**. Three
reasons:

1. The rail stays visible regardless of banner state.
2. Modals (`--z-modal` 1300) fully cover the banner (`--z-banner` 35) without leaving rail chrome poking through.
3. Branding stays consistent — the rail is always the "Azeer" surface.

## A11y

- L1 → `<nav aria-label="Primary navigation">`
- L2 → `<aside aria-label="Secondary navigation">`
- Main → `<main>`
- Details / folders / list panels in `objectDetail` → `<aside aria-label="…">` with the panel name

Five landmarks total in `objectDetail`, three in `standard` — Vox / NVDA /
JAWS users can jump between them with landmark-navigation shortcuts.

## Forbidden

- 4-zone variant — only `standard` (3-zone) or `objectDetail` (5-zone)
- Global topbar paired with AppShell — DS bans (L1 rail IS the chrome)
- Banner / HelpBubble / CallBar mounted inside a feature — shell slots only
- Resizing panel widths inline — if a feature needs more space, **collapse L1 or hide Details** instead
- Floating chrome positioned at the viewport from feature code — use shell slots
- Nested AppShells — single chassis per app
