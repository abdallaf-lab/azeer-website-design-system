## Anatomy

| Element | Spec |
| --- | --- |
| Track | 36 × 20 px, `rounded-full`, 2 px transparent border (reserves space for thumb travel) |
| Thumb | 16 × 16 px, `bg-surface` + `--shadow-control`, slides 16 px on toggle |
| Off state | Track = `border-strong` (gray) |
| On state | Track = `accent-fill` (indigo `#5238D1`) |

The track is the touch target (already wider than the 32 px WCAG floor) — no
extra hit-area padding required for mobile.

## Tokens used

| Token | Where |
| --- | --- |
| `--color-border-strong` | Off-state track |
| `--color-accent-fill` | On-state track |
| `--color-surface` | Thumb fill |
| `--shadow-control` | Thumb shadow (single tactile-control token) |
| `--radius-full` | Track + thumb shape |
| `--duration-fast` + `--ease-standard` | Color + transform transitions |
| `--focus-ring` (global) | Focus indicator — Switch carries `role="switch"`, covered by the global rule |

## States

| State | Mechanism |
| --- | --- |
| default (off) | Resting |
| hover | Subtle track darken (Radix doesn't add hover styling by default; pass `className="hover:bg-…"` if a hover signal is required) |
| focus | Global `:focus-visible` ring on the `role="switch"` element |
| checked | `data-[state=checked]:bg-accent-fill` on track + `translate-x-4` on thumb |
| disabled | `opacity: 0.5` + `cursor: not-allowed` |

## RTL behavior

Thumb travel direction mirrors via `rtl:data-[state=checked]:-translate-x-4`.
Flip the toolbar to RTL and the on-state thumb sits at the start side
instead of the end side. Track / thumb size are direction-agnostic.

## A11y

- Renders a `role="switch"` element. Radix handles `aria-checked` synchronization.
- Native `<input type="checkbox">` semantics are **not** used — `role="switch"` carries the on/off semantic (announced as "switch, on" / "switch, off" by screen readers, vs. "checkbox, checked").
- The track is wide enough (36 × 20 px) to act as a touch target on its own; pair with a visible label via `<Label htmlFor>` or wrap both in a `<label>` so click-target extends to the label text.
- Disabled with a recoverable cause: explain why via `FormField` `helper` — disabled-with-no-reason is hostile UX.

## Composition with FormField

`FormField` wires `aria-required` / `aria-describedby` onto the Switch. See
the **Inside FormField** and **Disabled — with reason** stories above.

## Forbidden

- `size` prop — there isn't one
- Switch + adjacent text without a `<label>` wrap or `htmlFor` binding — click-target is too small without the label extending it
- Custom track / thumb colors per feature — `accent-fill` on, `border-strong` off, period
- Switch for an action (Sign in, Save) — use `Button`. Switches are *state*, not *commands*.
- Disabling a Switch as a "loading" pattern — render in its current state, block the surrounding action through other means
