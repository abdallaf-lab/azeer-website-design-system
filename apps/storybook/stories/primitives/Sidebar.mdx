## Locked anatomy

| Property | Value |
| --- | --- |
| Width | **230 px** (L1 is 196 px; L2 sits 34 px wider so the L1→L2 transition reads as a clear step out into the workspace island) |
| Surface | White island, `rounded-xl`, 1 px `border-default` |
| Padding | `p-2` |
| Inter-item gap | `gap-1` |
| Title | Optional uppercase eyebrow (`text-label-xs text-fg-muted`) |
| Item height | 32 px (`h-8`) per Shell.md sidebar-row spec |
| z-index | `--z-nav` (20) |

## SidebarItem

Same active-state pattern as `PrimaryRailItem`:

- `accent-bg-subtle` background
- `accent-text` label
- 2 px start-edge bar in `accent-fill`

Optional `icon` (14 px), `trailing` slot (count, badge).

```tsx
<Sidebar title="Inbox">
  <SidebarItem icon={Inbox} label="Your inbox" active trailing="12" />
  <SidebarItem icon={AtSign} label="Mentions" trailing="3" />
  <SidebarItem icon={Star} label="Starred" />
</Sidebar>
```

For multiple titled sections within one Sidebar, render `<div>` groups
manually with `<div className="text-label-xs text-fg-muted px-2 py-1.5">`
eyebrows between groups (see **Multiple sections** story).

| Token | Where |

| `--color-fg-default` | Item label |

| `--radius-xl` (island) / `--radius-md` (items) | Shape tier |

## A11y

- `<aside aria-label="Secondary navigation">` landmark.
- Items use `aria-current="page"` when active.
- Focus ring from the global `:focus-visible` rule (Sidebar items are real `<button>`s).

## RTL behavior

- Sits at the inline-start side of Main (after L1 in LTR, before L1 mirror under RTL).
- Active 2 px bar uses logical `start-0` — mirrors automatically.

## Forbidden

- Sidebar width override — locked 230 px
- Sidebar without parent AppShell — works visually but the canvas / rounded-xl context is part of the chassis
- Mixed item sizes in one Sidebar — all `h-8` (32 px), period
- Sidebar as a primary navigation surface — L1 PrimaryRail is the primary; Sidebar is L2 only
