Minute-precision time entry. Thin wrapper around `<input type="time">` rendered
in Azeer Input chrome (border, focus tint, disabled / `aria-invalid`
plumbing, `tabular-nums` for HH:MM digit alignment). The browser hosts the
picker UI on focus — consistent across Chrome / Safari / Firefox modulo
locale, and native on iOS / Android (wheel pickers).

**A fully custom popover-based TimePicker is deferred to v1.2+** per
Components.md. This primitive is the v1.1 promotion — it ships the native
control with DS styling so consumers don't have to choose between
"DS-shaped" and "works on every platform".

## API

```tsx
const [time, setTime] = useState<string>("");

<TimePicker
  value={time}
  onValueChange={setTime}
  step={60}              // seconds, default 60 (minute resolution)
  min="09:00"
  max="17:00"
  size="md"
/>
```

| Prop | Type | Notes |
| --- | --- | --- |
| `value` / `defaultValue` | `string` | `HH:MM` (or `HH:MM:SS` when `step < 60`). Empty string when unset. |
| `onValueChange` | `(value: string) => void` | Fires on every change; receives the new time string |
| `step` | `number` | Step in seconds. Default `60` (minute). Set `1`–`59` to expose a seconds field. |
| `min` / `max` | `string` | Earliest / latest selectable. Format matches `value`. |
| `size` | `"sm" \| "md" \| "lg"` | Matches Input scale (32 / 40 / 48 px). |
| `disabled` | `boolean` | Disables the input. |
| `id` / `aria-*` | — | Pass-through for `FormField` integration. |

## Value format

24-hour clock, always:

| Step | Format | Example |
| --- | --- | --- |
| `60` (default) | `HH:MM` | `"14:30"` |
| `< 60` | `HH:MM:SS` | `"14:30:45"` |

The browser may **display** the time in 12-hour format depending on the
user's OS / locale — but the `value` you read and write is always 24-hour.
That's the spec for `<input type="time">`.

## Tokens used

Inherits the full Input token map (TimePicker is `<Input type="time">` with
`tabular-nums`):

| Token | Where |
| --- | --- |
| `--color-surface` / `-border-strong` | Input surface + border |
| `--color-border-focus` | Focus border tint |
| `--color-fg-default` / `-fg-subtle` | Value text / placeholder |
| `--color-danger-border` / `-danger-fill` | `aria-invalid` border |
| `--text-body-sm` / `-body-md` | Per size |
| `--radius-md` | Corner radius |

## Pattern: combine with a date input

For full datetime entry, compose a date input + TimePicker inside a
`FieldGroup` instead of using a single `<input type="datetime-local">` —
this gives the DS-styled trigger pair and keeps date and time values
independently controllable. See **Inside FieldGroup with DatePicker** above.

> The native `<input type="datetime-local">` is intentionally **not** wrapped
> in this batch — its browser UI varies more, and most real flows want
> separate date and time controls anyway (different validation, different
> default values, different RTL behavior).

## A11y

- Real `<input type="time">` — full keyboard and screen-reader support from the platform.
- Inside `FormField`: `label`, `aria-required`, `aria-describedby`, and `aria-invalid` are wired automatically.
- `Tab` moves focus into the input. Inside the picker, arrow keys adjust hour / minute / period segments — native browser behavior.
- `Esc` closes the open picker without committing in browsers that show a popup.

## RTL behavior

- The input itself stays LTR for HH:MM (digits are always Latin per Numerals.md).
- Layout — surrounding label / helper / icon position — mirrors normally because the Input wrapper uses logical properties.
- Inside RTL prose, place the TimePicker on its own line or wrap inline usage in a `bidi-isolate` span.

## Forbidden

- Custom 12-hour AM/PM toggle — let the browser locale handle display format. The value contract is always 24-hour.
- Date-time entry via a single `<input type="datetime-local">` wrapped here — compose a date control + `TimePicker` inside `FieldGroup` instead.
- Hour-only or minute-only "spinner" entry as a TimePicker variant — that's a `NumberInput` with bounds, semantically.
- Arabic-Indic numerals in the displayed time — DS-locked to Latin per Numerals.md `numberingSystem: 'latn'`.
- Using TimePicker for *duration* entry (e.g. "1h 30m") — duration is a different primitive (`NumberInput` pair, or future `DurationInput`); TimePicker represents a point in time, not an interval.
