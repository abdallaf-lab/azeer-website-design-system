# Motion

**CSS-only tokens.** Three durations, four easings. Framer Motion is feature-level
only — never in the design-system layer. Composed transition shorthands prevent
duration / easing drift.

## Composed shorthands

<Callout intent="info" title="Always reach for the shorthand">
Components transition via the shorthand, not the parts — prevents duration /
easing pairs from drifting apart.

```css
/* ✓ */ .button { transition: background-color var(--transition-fast); }
/* ✗ */ .button { transition: background-color 120ms cubic-bezier(0.4, 0, 0.2, 1); }
```
</Callout>

## Reduced motion

<Callout intent="success" title="Built into the token layer">
`@media (prefers-reduced-motion: reduce)` collapses all three duration tokens
to 0ms — components don't branch, transitions still wire up, they just resolve
instantly. A global CSS rule also disables all keyframe animations.
</Callout>

## Forbidden

- Animation durations outside 120 / 200 / 280 ms
- Easings outside `--ease-out / -in / -standard / -emphasized`
- Motion longer than 320ms (except `--ease-emphasized` on opt-in AI surfaces)
- Importing Framer Motion inside `@azeer/ui`
- Animating `box-shadow` directly (performance) — transition opacity on a pseudo-element instead
- Motion that bypasses the token system (so reduced-motion can't disable it)
