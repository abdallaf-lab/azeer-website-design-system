# Theming

**Three independent axes, all set on `<html>`.** v1 ships `light` +
`comfortable` + both directions. Other values are **scaffolded but not
implemented** — adding them means defining the primitive-layer overrides,
zero component changes.

<ThemingAxesTable />

## How a theme switch works

```css
/* tokens.css */

:root {
  /* Primitive layer (light theme — current default) */
  --neutral-canvas:  #F4F3FB;
  --neutral-default: #2A2A29;
}

[data-theme="dark"] {
  /* Re-define ONLY the primitives — semantic layer references them */
  --neutral-canvas:  #0F0E16;
  --neutral-default: #F4F3FB;
}
```

```html
<html data-theme="dark" data-density="comfortable" dir="ltr">
```

Every Tailwind utility (`bg-surface`, `text-fg-default`, …) reads through
`@theme inline { … }` which references primitive vars. Flip the attribute on
`<html>` and every component switches with zero rebuild.

<Callout intent="info" title="Forbidden alternatives">
- `@media (prefers-color-scheme: dark)` — theming is attribute-driven, not media-query-driven
- A `.dark` class selector — use `[data-theme="dark"]` when it ships
- Adding `dark:` / `compact:` variants in components today (the values don't exist in v1)
</Callout>

## RTL enforcement (shipped in v1)

The direction axis is the only one fully active in v1. Logical CSS utilities
auto-mirror; physical utilities are **banned via ESLint** (already wired in
`@azeer/config-eslint/react`).

```tsx
/* ✓ logical — auto-mirrors */
<div className="ms-2 ps-3 border-s rounded-s text-start" />

/* ✗ physical — ESLint error */
<div className="ml-2 pl-3 border-l rounded-l text-left" />
```

See **Foundation / RTL** in the spec (and the upcoming RTL story under
Components) for what stays LTR even inside RTL prose — phone numbers, IDs,
emails, code blocks, kbd shortcuts, brand wordmarks, time-series charts.
