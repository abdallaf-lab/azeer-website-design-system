# Typography

**Pixel-snapped 9-tier ramp + 1 display.** Inter for Latin, IBM Plex Arabic
for RTL, JetBrains Mono for code / IDs / phone numbers. Same-metric pairing
so mixed prose doesn't wobble. Three weights only: 400 / 500 / 600.

<Callout intent="info" title="Why pixel-snapped, not a mathematical ratio">
Inter is hinted for whole-pixel rendering. Subpixel sizes (e.g. 14.4px from a
1.125 ratio) blur inconsistently across browsers. We use 1px steps under 14px
(high JND at small sizes), 2px steps mid-range, 4px jump to display.
</Callout>

## Where each token goes

| Token | Size / LH | Use |
| --- | --- | --- |
| `text-label-xs` | 11 / 14 / 600 / +0.05em UPPER | UPPERCASE eyebrows — Inspector section titles, sidebar group labels |
| `text-body-xs` | 12 / 16 / 400 / +0.01em | Metadata, captions, timestamps, footnotes only — **not general body** |
| `text-label-sm` | 13 / 16 / 500 | Form labels |
| `text-body-sm` | 13 / 20 / 400 / +0.005em | Compact body, dense rows |
| `text-label-md` | 14 / 16 / 600 | Button labels (all sizes), emphasized form labels |
| `text-body-md` | 14 / 20 / 400 | **★ DEFAULT** body, default control text |
| `text-heading-sm` | 16 / 22 / 600 / -0.01em | Card title, list-column title |
| `text-heading-md` | 18 / 24 / 600 / -0.015em | Sub-nav / panel title |
| `text-heading-lg` | 20 / 28 / 600 / -0.018em | Page header, module title |
| `text-heading-xl` | 24 / 32 / 600 / -0.022em | Hero in app (dashboard title, settings landing) |
| `text-display` | 32 / 40 / 700 / -0.03em | True display tier — empty-state heroes, marketing surfaces |

## Decision tree — label vs body

| Need | Use |
| --- | --- |
| UI affordance label (button, form field label, eyebrow) | `text-label-*` (heavier weight, tighter LH, designed for fixed-height controls) |
| Content tier (paragraphs, list rows, descriptions) | `text-body-*` (regular weight, generous LH, designed for reading) |

If two tokens share a pixel size (e.g. `label-sm` and `body-sm` are both 13px), they differ in **weight and line-height** because they serve different roles. Pick by intent, not by size.

## Density behavior

Type sizes do **not** change under `data-density="compact"` — density is content-density, not type-density. Compact flips control heights and row heights only; legibility stays constant across modes.
