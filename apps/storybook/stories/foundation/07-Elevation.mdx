# Elevation

**Borders-first.** A 1px border defines a resting surface; shadows are reserved
for floating layers that escape document flow. Four tiers — never add a fifth.

<ElevationScale />

## Component → tier mapping

| Surface | Tier |
| --- | --- |
| Cards (resting), shell panels, list rows | `--elev-1` (border only) |
| Popover, DropdownMenu, ContextMenu, HoverCard, Tooltip | `--elev-2` |
| Dialog, Sheet, Toast, HelpBubble | `--elev-3` |
| AI Summary on hover | `--elev-ai` *(only place the brand-tinted shadow appears)* |
| Switch / Slider thumb | `--shadow-control` |

## Focus ring — single shorthand

<Callout intent="info" title="Components NEVER re-roll the focus ring">
The focus ring is conceptually elevation — a visual signal applied above a
surface. One token, one global selector, one source of truth. If a parent's
`overflow: hidden` clips the outline, swap that element to the `box-shadow`
form — but the token values stay identical.
</Callout>

<FocusRingTable />

The global rule (already wired in `tokens.css`):

```css
:where(button, a, input, select, textarea, [role="button"], [tabindex]):focus-visible {
  outline: var(--focus-ring);
  outline-offset: var(--focus-ring-offset);
  border-radius: inherit;
}
```

## Alpha scale

State overlays, dividers, scrims, and tinted shadows pull from this scale so
density modes can scale them in one place.

<AlphaTable />

## Forbidden

- Shadow on a resting card (use border via `--elev-1`)
- Tailwind defaults `shadow-sm/md/lg/xl/2xl`
- Inline `style={{ boxShadow: '…' }}`
- Adding a 5th elevation tier
- Re-rolling the focus ring per-component
- Using `--elev-ai` anywhere outside AI surfaces
