# Colors

Every color a component can touch. Primitive scales live underneath; this page
shows the **consumer surface** — purpose-named semantic tokens with verified
contrast pairs.

<Callout intent="info" title="Three-tier model">
**Primitive** (indigo-50…950, neutrals, intent hexes) → **Semantic**
(`--color-accent-fill`, `--color-fg-default`) → **Theme bridge** (Tailwind
`@theme` generates utilities). Components consume tier 2.
</Callout>

<Callout intent="warning" title="Brand vs functional fill">
`#7B61FF` (`--color-accent-brand`) is the locked brand identity and clears
4.14:1 — passes for non-text UI (focus rings, illustrations). For white-text
CTAs we use `--color-accent-fill` (`#5238D1`, 7.3:1 AAA). Both belong to the
same indigo family. Never put white text on `--color-accent-brand`.
</Callout>

<Callout intent="danger" title="Warning fill is icon-only">
`--color-warning-fill` (`#F2B232`) fails 2.0:1 against white. Use
`warning-bg-subtle` + `warning-text` for badges/alerts. Reserve the fill
for icons and progress segments.
</Callout>
